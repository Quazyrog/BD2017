Bazy Danych 2017 – Projekt Zaliczeniowy

W ramach projektu ma powstać baza danych do analizy logów serwera WWW. Program ma umożliwiać wczytanie tekstowego pliku logu do tabel bazodanowych zaprojektowanych w modelu relacyjnym. Załadowane z logu informacje powinny być następnie udostępnione do analizy przez stworzony na użytek projektu interfejs użytkownika. Baza danych powinna zostać zaimplementowana na serwerze Oracle. Interfejs może być zrealizowany w PHP lub w innym języku umożliwiającym łączenie się z bazą danych Oracle. Część przetwarzania powinna zostać umieszczona na serwerze danych – w postaci kodu PL/SQL.

Proces ładowania logu może być dostępny z interfejsu użytkownika lub jako funkcja uruchamiana wsadowo. Należy rozważyć i rozwiązać problemy związane z ładowaniem kolejnych logów lub ponownym ładowaniem tego samego pliku.

Ciekawe formaty:

%h

Remote host

%H

Request protocol

%r

First request line

%[>]s

Status (generalnie status pierwszej odpowiedzi, bez przekierowań; %s oznacza ostatni status)

%b

Rozmiar bez nagłówków (- oznacza 0)

%B

Rozmiar z nagłówkami

%A

Local IP address

%{format}t

Czas; można podać format jak w strftime

%D / %T / {unit}T

Time to serve in microseconds / miliseconds / podanej jednostce (us/ms/s)

%m

Metoda

%R

Handler generating response

%U

URL path without query string

%q

The query string (prepended with a ? if a query string exists, otherwise an empty string)

%v

The canonical ServerName of the server serving the request.

%V

The server name according to the UseCanonicalName setting.

Co chcemy mieć:

  • Trzymanie logów z różnych serwerów i zarządzanie tymi serwerami

  • Trzymanie informacji o tym, co się dzieje jak wgrywamy (w sensie przywiązanie wpisów do konkretnej operacji wgrywania logu)

  • Wykrywanie duplikatów przy ładowniu

  • Podawanie jaki ma być format

  • Wyciąganie %m %U %q z %r

  • Zawansowanie wyszukiwanie z możliwościa agregowania sensownych pól

  • Tworzenie filtrów

  • Lista paskud, być może ich grupowanie

  • ładne wykresiki z tych wiltrów i też tworzone w tabelkach chyba nie

interface LogEntry {
    (key id)

    attribute int id;

    attribute int size;
    attribute string remoteAddress;
    attribute date time;
    attribute string handler;

    attribute string request;
    attribute string protocol;
    attribute string urlPath;
    attribute string queryString;

    reference LogFile uploadedFrom;
}

interface LogFile {
    (key id)

    attribute int id;
    attribute date uploadDate;
    attribute string uploadFormat;
    attribute int duplicatesSkipped;
    attribute string note;

    reference Server server;
}

interface Server {
    (key name)

    attribute string name;
    attribute string description;
    attribute string defaultLogFormat;
}


interface Filter {
    (key name)

    attribute string name;
    attribute string queryString;
}

interface FilterList {
    (key name)

    attribute string name;
    attribute Set<string> values;
}
